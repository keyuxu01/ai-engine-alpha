# Tool 调用与统一通信协议 （为什么会需要 MCP）

## 背景：Tool 扩展了 AI 的能力

- 原本：大模型只能思考，不能做事情
- 现在：可以通过调用 Tool 来实际执行任务

## 问题：跨语言调用困难

如果 AI Agent 是用 Node.js 写的，但你的工具是 Java、Python、Rust 等其他语言写的，怎么调用？

### 解决方案 1：stdio（标准输入输出）

- 单独开一个子进程来运行其他语言写的代码
- 通过标准输入输出流进行通信

### 解决方案 2：HTTP

- 在本地跑一个 HTTP 服务
- 通过 HTTP 请求进行调用

## 核心问题：协议不统一

每个人实现的方式都不一样：

- A 用 stdio
- B 用 HTTP
- C 用 WebSocket
- ...

**结果：** 每次要接入新工具，都要先去了解它是怎么定义的（接口格式、参数、返回值等），非常麻烦！

## 解决思路：统一通信协议

定义一个**统一的通信协议**，大家都按同样的格式来沟通，这样无论什么语言写的工具，只要遵循这个协议就能直接接入。

### 统一协议的好处

**请求格式：**

```json
{
  "name": "get_weather",
  "arguments": { "city": "北京" }
}
```

**响应格式：**

```json
{
  "result": "晴，25°C"
}
```

只要都遵循这个格式：

- Python 写的天气服务 ✓
- Node.js 写的数据库工具 ✓
- Rust 写的文件处理 ✓

前端统一调用，不需要关心背后是什么语言！

---

**这就是 MCP（Model Context Protocol）做的事情**——它就是那个"统一的通信协议"

## MCP 的传输方式

MCP 最大的特点就是可以跨进程调用工具：

- **跨本地进程**调用 → 用 **stdio**（标准输入输出）
- **跨远程进程**调用 → 用 **HTTP**

ai agent 就是 MCP 客户端，可以通过 MCP 协议调用各种 MCP Server，实现跨进程的工具调用。
